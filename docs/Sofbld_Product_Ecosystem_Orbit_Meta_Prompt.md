# Sofbld Product Ecosystem Orbit Meta Prompt

## Purpose

Use this document as a high-fidelity meta prompt for designing Sofbld's hero replacement for `/src/assets/Interface.jpg`. The goal is not to generate generic 3D "tech visuals." The goal is to direct a designer, 3D artist, creative technologist, or AI design tool toward a premium hero concept that feels strategically aligned with Sofbld's market, visually differentiated, and executed to a best-in-class product standard.

This prompt is intentionally detailed so the resulting work reflects strong product thinking, not just attractive rendering.

---

## Project Context

Sofbld is positioning itself as a premium software consulting and engineering partner for businesses that need custom systems rather than templates.

The current homepage messaging establishes that Sofbld builds:

- web applications
- mobile apps
- AI automations and embedded AI workflows
- internal platforms and operational tools
- specialized financial and quantitative software

The hero copy currently communicates:

- "Custom software systems for businesses that need more than templates."
- Sofbld serves serious businesses and founders
- Sofbld handles both mainstream and technically advanced projects
- Sofbld should feel premium, technically credible, high-trust, and outcomes-focused

This visual must support that positioning immediately.

Reference context:

- Current hero implementation: `src/components/Hero.jsx`
- Current hero styling and visual stage: `src/components/Hero.css`
- Brand and conversion strategy: `docs/Softbuild_Conversion_Audit.md`

## Core Creative Direction

The selected concept direction is:

**Product Ecosystem Orbit**

This concept should present Sofbld not as a company that ships isolated deliverables, but as a firm that architects complete digital ecosystems. The scene should imply that multiple product surfaces, automation layers, intelligence systems, and data tools can coexist elegantly because they are all orchestrated through disciplined engineering.

This is not a collage of screens.

This is not a spinning gadget.

This is not a sci-fi object for its own sake.

It is a controlled, premium, living system.

---

## Style Synthesis

This concept should combine three visual style variants in a disciplined hierarchy:

### 1. Boardroom Luxury
This is the dominant visual base.

Use it for:

- materials
- composition
- restraint
- negative space
- premium atmosphere
- confidence and trust

The scene should feel like it belongs in a high-end boardroom presentation, private banking interface, or premium enterprise launch environment.

### 2. Systems Intelligence
This is the dominant behavior and motion language.

Use it for:

- connective logic
- orchestration
- signal flow
- responsive state changes
- active system behavior
- the sense that the ecosystem is alive, coordinated, and operating in real time

### 3. Fintech Precision
This is the detail and geometry layer.

Use it for:

- edge definition
- structural rigor
- contour accents
- disciplined curves
- graph-like surfaces
- exactness and high-trust technical seriousness

### Blend Ratio

Use this weighting as a design guardrail:

- 55% Boardroom Luxury
- 30% Systems Intelligence
- 15% Fintech Precision

This balance matters. The scene must feel premium first, intelligent second, and precise third. If the fintech layer dominates, the brand will look too narrow. If the systems layer dominates, it risks looking like generic AI imagery. If the luxury layer dominates without enough intelligence, it risks looking decorative rather than capable.

## Concept Statement

Create a hero scene centered on a refined technical core surrounded by a small number of orbiting product forms. Each orbiting form represents a category of digital capability that Sofbld builds. Thin, elegant connection paths and signal behavior reveal that these are not disconnected deliverables. They are parts of one engineered ecosystem.

The emotional effect should be:

- composed
- elevated
- intelligent
- exact
- calm under complexity

The business effect should be:

- increased trust
- higher perceived project sophistication
- clearer alignment with high-value custom software work
- stronger premium positioning

## Scene Architecture

Design one hero scene with one clear focal system.

### Central Core

The central core should represent Sofbld's engineering intelligence and system architecture capability.

It should feel:

- sculptural
- machined
- composed
- technically elegant
- dimensional without becoming bulky
- abstract enough to be versatile
- structured enough to feel intentional

The central core must be the visual anchor. Everything else should reinforce it.

Potential characteristics:

- layered architectural planes
- smoked-glass volumes
- metallic rails or framing
- precision-cut seams
- soft internal illumination
- controlled contour detail
- subtle active data pulses

### Orbiting Elements

Use 3 to 4 orbiting elements only. More than that will dilute clarity and reduce the premium feel.

Each orbiting element should feel like a refined product form, not a literal screenshot slapped into 3D space.

Recommended orbiting set:

- a web application plane
- a mobile device slab
- an AI intelligence cluster
- a quant/data ribbon or precision surface

### Internal Tools Representation

Do not add a separate oversized internal-tools object unless there is an exceptionally elegant reason.

Instead, express internal tools and automation through:

- system connectors
- workflow nodes
- orchestration paths
- operational signal routing
- subtle modular transitions between elements

That approach keeps the hero cohesive while still representing internal platforms and automation capability.

### Relationship Logic

Every object should clearly feel related to the whole. The scene should imply:

- shared architecture
- common engineering logic
- interoperability
- data flow
- system ownership
- product ecosystem design

Nothing should look like an isolated marketing icon floating in empty space.

## Meaning of Each Visual Element

### Web Application Plane

This should suggest:

- SaaS interfaces
- dashboards
- operational platforms
- portals
- business workflows

Visual cues may include:

- layered panel geometry
- modular blocks
- subtle chart scaffolding
- interface framing
- structured card-like depth

Keep it elegant and abstracted. Avoid turning it into a literal dashboard screenshot in 3D unless the execution is exceptionally refined.

### Mobile Device Slab

This should suggest:

- customer-facing apps
- field tools
- mobile operations
- companion interfaces

Visual cues may include:

- ultra-thin edge treatment
- premium device proportions
- edge lighting
- minimal interface depth
- subtle gesture or navigation cues

It should feel integrated into the scene, not like a generic phone mockup.

### AI Intelligence Cluster

This should suggest:

- embedded intelligence
- inference layers
- workflow automation
- knowledge systems
- signal-based reasoning

Visual cues may include:

- luminous node groupings
- translucent neural-like structure
- disciplined branching logic
- responsive data illumination
- subtle adaptive movement

Avoid cliché AI tropes. No glowing brain icons. No cartoon networks. No chaotic neural spaghetti.

### Quant or Data Ribbon

This should suggest:

- financial tooling
- research interfaces
- analytics depth
- high-precision data systems
- complex but controlled computation

Visual cues may include:

- contour lines
- disciplined wave surfaces
- signal ribbons
- structured topography
- graph-like edge articulation

This layer adds seriousness and technical distinction. It should not overpower the broader brand.

## Composition Principles

The scene should be composed like premium product cinematography.

Key principles:

- one dominant focal point
- clean silhouette from a distance
- clear depth separation
- asymmetry with balance
- generous negative space
- controlled light hierarchy
- premium restraint

The composition should feel expensive before the viewer notices any individual detail.

Avoid clutter. Avoid trying to represent every service equally. The system should feel curated, not exhaustive.

---

## Material Language

Materials should communicate trust, sophistication, and precision.

Primary material families:

- smoked glass
- frosted translucent surfaces
- graphite metal
- brushed dark alloys
- satin technical finishes
- restrained emissive highlights

Secondary material families:

- soft illuminated seams
- faint holographic contour overlays
- low-opacity technical panels
- carefully controlled reflective accents

Material goals:

- tactile without becoming heavy
- futuristic without becoming sci-fi
- premium without becoming flashy
- technical without becoming cold

Do not use materials that feel plasticky, toy-like, or over-rendered.

---

## Color Direction

The color system should be highly disciplined.

Base palette:

- deep navy
- graphite
- midnight blue-black
- smoked steel

Primary light accent:

- cool electric blue or cyan-blue

Optional support accent:

- very restrained warm gold or amber

Color rules:

- use glow sparingly
- reserve high saturation for meaningful focal points
- keep most surfaces in dark neutral tones
- let light guide the eye rather than flooding the scene

The scene must feel premium and calm, not neon, loud, or gaming-oriented.

---

## Lighting Direction

Lighting should do significant narrative work.

Use lighting to communicate:

- confidence
- precision
- dimensionality
- selective emphasis
- environmental atmosphere

Preferred qualities:

- cinematic rim lighting
- low-key ambient fill
- localized bloom only on important signals
- layered depth lighting
- soft reflections with controlled intensity

The lighting should make the object feel alive without looking over-processed.

Avoid:

- blown-out bloom
- aggressive lens flare
- candy-colored glow fields
- noisy volumetric effects

---

## Motion Direction

Motion must feel deliberate and premium.

Think:

- controlled drift
- orbital grace
- system synchronization
- signal pulses
- subtle state transitions
- calm power

Do not think:

- spinning showcase object
- hyperactive gaming menu
- constant particle storm
- random idle animation

### Motion Hierarchy

Use a layered motion system:

1. Ambient motion
Slow breathing, drift, and depth movement that makes the scene feel alive.

2. Orbital motion
Orbiting product forms should move subtly and on different timing curves.

3. System signal motion
Connections and nodes can pulse with faint data movement.

4. Focus motion
On hover, pointer movement, or section activation, certain elements can become slightly more present.

### Motion Tone

The scene should feel:

- confident
- precise
- expensive
- measured

If the user can immediately describe the animation as "cool," that is not enough. The better reaction is:

"This feels sophisticated."

---

## Interaction Design

Interactivity should be subtle and product-grade.

### Pointer Behavior

Allow slight parallax and perspective response based on pointer position, but keep it restrained. The motion should feel like premium physical depth, not a playful toy.

### Scroll Behavior

Use scroll to gently shift emphasis rather than fully transform the scene into something unrelated.

Possible effects:

- light angle changes
- activation of specific orbit elements
- signal pulses intensify briefly
- camera framing shifts slightly
- one service surface becomes subtly more readable

### Section Awareness

If later connected to service sections, the same master scene can express service-linked state changes:

- Web: interface plane becomes slightly more active
- Mobile: device slab becomes more legible
- AI: node cluster glows or reorganizes
- Quant: precision ribbon sharpens or brightens
- Internal Tools: connector paths and orchestration behavior intensify

The master scene should remain one integrated system throughout.

## Design Process Requirements

Follow this process in order. Do not jump directly into styling before strategic clarity is established.

### Phase 1. Brand Calibration

Before designing, confirm the visual is aligned with:

- Sofbld's positioning as a software consulting and engineering partner
- the current hero message
- premium pricing aspirations
- audience expectations for trust and competence
- service breadth without visual confusion

Questions to answer internally:

- What must the viewer believe within 3 seconds?
- What signals premium consulting instead of agency fluff?
- What signals product engineering depth instead of generic "tech"?
- What details could accidentally narrow the brand too far into fintech or AI?

### Phase 2. Meaning Mapping

Map each visual element to a business meaning.

No element should exist only because it looks attractive.

For each object or motion behavior, define:

- what it represents
- why it belongs
- what buyer assumption it is intended to create
- whether that assumption strengthens Sofbld's market position

### Phase 3. Silhouette and Composition Exploration

Explore multiple composition routes before polishing.

Investigate:

- central-core dominant compositions
- diagonal orbit paths
- front-biased versus angled camera views
- tighter versus more spacious framing
- low orbit count versus moderate orbit count

Evaluate each route for:

- clarity at a glance
- premium feel
- immediate perceived sophistication
- mobile adaptability
- ability to coexist with headline and CTA copy

### Phase 4. Material and Lighting Studies

Once the form language is chosen, run material studies.

Test:

- darker matte metals versus softer satin metals
- cleaner glass versus more atmospheric haze
- cooler overall scene versus controlled warm highlights
- stronger edge illumination versus subtler seam lighting

Choose the combination that feels the most elevated and least trendy.

### Phase 5. Motion Prototyping

Test motion separately from visual polish.

Ask:

- does the motion add confidence or distraction?
- does it make the scene feel alive or merely animated?
- does it support trust?
- does it respect premium restraint?

Motion should clarify the system and strengthen the narrative, not just decorate it.

### Phase 6. Responsive Strategy

Design desktop, tablet, and mobile intentionally.

Do not assume the same full scene should appear unchanged on all devices.

Define:

- the full desktop experience
- a lighter tablet variant
- a simplified mobile composition or graceful fallback

The mobile version should preserve elegance and meaning even if much of the scene is reduced.

### Phase 7. Accessibility and Comfort Review

Check for:

- motion sensitivity risk
- visual overload
- readability competition with headline text
- contrast issues
- clutter near CTA elements

Premium design must feel effortless, not demanding.

### Phase 8. Product Quality Review

Conduct a final critique against strategy, craft, performance, and usability before approval.

If the scene is beautiful but unclear, it is not done.

If it is impressive but generic, it is not done.

If it is premium but static and emotionally cold, it is not done.

If it is creative but distracts from conversion, it is not done.

- suggests ecosystem thinking rather than single-project output
- balances beauty with business clarity
- contains a strong focal point
- maintains visual hierarchy

## Failure Modes to Avoid

Do not produce any of the following:

- generic glowing orb visuals
- random floating glass panels with no relationship logic
- cyberpunk neon overload
- overuse of purple or rainbow gradients
- too many orbiting objects
- obvious stock-dashboard aesthetics
- literal phone mockups that feel like app-store ads
- AI visuals that look like marketing cliché
- fintech visuals so strong that the brand feels finance-only
- motion that looks like a product demo reel instead of a premium hero
- anything that makes the headline feel secondary

---

## Performance and Practicality Standards

Even at the concept stage, design with implementation reality in mind.

The concept should anticipate:

- one strong hero scene rather than many scenes
- controlled geometry complexity
- limited post-processing
- careful use of transparency
- optimized textures if any are required
- non-essential motion being reduced on lower-power devices
- reduced-motion support
- a version that still looks polished when 3D is simplified

The correct premium outcome is not the heaviest scene. It is the most convincing scene per unit of complexity.
